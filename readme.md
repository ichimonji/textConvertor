# 汎用テキストコンバータ

「Original Text」欄に変換したいテキストを入れ各種の変換ボタンを押してください。「Converted Text」欄に変換後のテキストが出力されます。  
変換後のテキストをさらに変換したい場合は「MOVE」ボタンを押して変換後テキストを「Original Text」欄に移動させてください。  
「AUTO MOVE」にチェックを入れると自動で変換後のテキストが「Original Text」欄に移動します。

「Original Text」欄でテキストを選択したまま各種変換ボタンを押すと、その選択範囲にのみ変換が適用されます。  
「DELETE」でテキストがすべて削除されます。  
「UNDO」を押す度に変換前の状態に一つずつ戻ります。  
「REPLACE」を押すとテキストエリアの左右を入れ替えることができます。

項目が多すぎて扱いづらい場合は「ケース」、「Url/Base64」といったグループ名の部分を押すとそのグループが折りたたまれます。  
折りたたまれたグループはもう一回押すと元に戻ります。

## 各項目の説明

### コード/数値型

#### ケース

半角英単語のケース操作を行う。またケース変換の他にクォーテーションの変更や符号付ラテン文字の符号解除なども行う。

+ `TITLE`  
  「タイトルケース」 頭文字のみ大文字に
+ `UPPER`  
  「アッパーケース」 全て大文字に
+ `LOWER`  
  「ロウアーケース」 全て小文字に
+ `SWAP`  
  「ケーススワップ」 大文字小文字を逆転
+ `\'\'→\"\"`  
  シングルクォーテーションをダブルクーテーションに
+ `\"\"→\'\'`  
  ダブルクーテーションをシングルクォーテーションに
+ `abc-def→abc_def`  
  文字間のハイフンをアンダーバーに
+ `abc_def→abc-def`  
  文字間のアンダーバーをハイフンに
+ `abcDef→abc-def`  
  キャメルケースをハイフンに
+ `abc-def→abcDef`  
  文字間のハイフンをキャメルケースに
+ `Àóṫœ→Aoto`  
  符号付きラテン文字の符号を解除、また合字も非合字化

※ 「符号付きラテン文字を標準化」では結合分音記号も削除します。

#### エンコード

各種エンコード／デコードを行う。

+ `URL エンコード(軽)`  
  パーセントエンコード（;/?:@& = +$-\_!~\*.,()a\#\'以外）を実行
+ `URL エンコード(中)`  
  パーセントエンコード（\-\_!~\*.()a\'以外）を実行
+ `URL エンコード(強)`  
  パーセントエンコード（\*+-\_./以外）を実行
+ `URL デコード`  
  パーセントエンコードをすべてデコード
+ `BASE64 エンコード`  
  BASE64エンコードを実行
+ `BASE64 デコード`  
  BASE64エンコードをすべてデコード

#### エスケープ

Unicode関連の変換を行う。

+ `エスケープ`  
  Unicode値でのパーセントエンコード（%u～）
+ `アンエスケープ`  
  Unicode値でのパーセントデコード
+ `HTMLエスケープ(中)`  
  HTML数値文字参照（\&#x～;）
+ `HTMLエスケープ(強)`  
  全文字をHTML数値文字参照（\&#x～;）
+ `HTMLアンエスケープ`  
  文字参照、実体参照を復号
+ `[<>\&]の実体参照化`  
  HTMLタグ用文字を実体参照化
+ `UTF-32→UTF-16`  
  UTF32のオーバーフロー分をサロゲートペアに変換
+ `UTF-16→UTF-32`  
  サロゲートペアをUTF32表記に
+ `JSエスケープ`  
  Javascript用に特殊文字をエスケープします。  
  エスケープされるのは改行（\\n,\\r）、タブ（\\t）、垂直タブ（\\v）、改ページ（\\f）、およびバックスラッシュ（\\）です。
+ `JSアンエスケープ`  
  上記のエスケープ文字をすべて元に戻します。

#### 数値型

数値型に対しての数々の変換を行う。

+ `Dec→Bin` `Bin→Dec`： 10進数を2進数に、あるいはその逆に変換します。
+ `Dec→Hex` `Hex→Dec`： 10進数を16進数に、あるいはその逆に変換します。
+ `Bin→Hex` `Hex→Bin`： 2進数を16進数に、あるいはその逆に変換します。
+ `1000→1,000` `1,000→1000`： 数字を3桁カンマ区切りに、あるいはその逆に変換します。
+ `Abs`： テキスト中のすべての数値を絶対数化します。
+ `Int`： テキスト中のすべての数値を整数化します。
+ `Round`： テキスト中のすべての数値を四捨五入します。
+ `Floor`： テキスト中のすべての数値を切り捨てます。
+ `Ceil`： テキスト中のすべての数値を切り上げます。

※全角の数字にたいしては無視します。

### 言語

#### 数字

アラビア数字（算用数字）からほかの数字記号への変換などを行う。

+ `123→百二十三`： アラビア数字を桁付き漢数字に変換します。
+ `123→一二三`： アラビア数字を桁なしの漢数字に変換します。
+ `百二十三→123`： 漢数字をアラビア数字に変換します。
+ `Arabic→Roman` `Roman→Arabic`  
  アラビア数字をローマ数字に或いはその逆に変換に変換します。  
  変換型は「M(1~3999)」型、「ↀ(1~399999)」型、「CIↃ(1~399999)」型の3つから選ぶことができます。

※ アラビア数字をローマ数字に関して、ↀ型の場合の100000である「ↈ（&amp;#x2188;）」はほとんどのフォントで表示が不可能です。  
※ アラビア数字から漢数字の変換（桁付き）に関して、少数の桁（分、厘、毛など）に関しては考慮せず数字のみを列記する方式で変換します。

#### 日本語関連

日本語テキストに関して様々な変換を行う。

+ `あ→ア` `ア→あ`： 全角ひらがなを全角カタカナに、あるいはその逆に変換します。
+ `ｱ→ア` `ア→ｱ`： 半角カタカナを全角カタカナに、あるいはその逆に変換します。
+ `A1→Ａ１` `Ａ１→A1`： 半角英数字を全角英数字に、あるいはその逆に変換します。
+ `新字→旧字（中）`： 新字体を旧字体に（異体字セレクタ分を含まない）に変換します。
+ `新字→旧字（強）`： 新字体を旧字体に（異体字セレクタ分を含む）に変換します。
+ `旧字→新字`： 旧字体を新字体に（異体字セレクタ分を含む）に変換します。
+ `一二→壱弐` `壱弐→一二`： 漢数字を大字に、あるいはその逆変換します。
+ `トウキョウ→tôkyô`： 全角カタカナを訓令式ローマ字表記に変換します。
+ `トウキョウ→tokyo`： 全角カタカナをヘボン式ローマ字表記に変換します。
+ `tokyo→トキョ`： ヘボン式ローマ字を全角カタカナに変換します。

※ 全角カタカナからの訓令式・ヘボン式ローマ字変換に関して、特殊な用法や表外の訓に関しては（99式などに倣った）独自の変換をしています。  
※ 新字体→旧字体の変換において旧字が別字として存在しない、あるいはUTF16範囲にない場合は異体字セレクタによって旧字体を表示します。

#### CJK関連

漢字に対して様々な変換を行う。

+ `日→簡体`  
  日本字体を簡体字に
+ `日→繁体`  
  日本字体を繁体字に
+ `簡体→日`  
  簡体字を日本字体に
+ `繁体→日`  
  繁体字を日本字体に
+ `簡体→繁体`  
  簡体字を繁体字に
+ `繁体→簡体`  
  繁体字を簡体字に
+ `ピンイン`  
  漢字の直後にピンインを追加

※ CJK変換全般に関して、一部の文字の変換が不可逆変換であるため正しく変換されない恐れがあります。  
※ ピンイン表記追加に関して、ピンインは「[A/B/C]」といった形式で表示します。

#### Sanskrit

サンスクリットの各種変換を行う。

+ `संस्कृत→saMskRta`  
  デーヴァナガリーを京都・ハーバード方式に
+ `संस्कृत→saṃskṛta`  
  デーヴァナガリーをIASTに
+ `saMskRta→संस्कृत`  
  京都・ハーバード方式をデーヴァナガリーに
+ `saMskRta→saṃskṛta`  
  京都・ハーバード方式をIASTに
+ `saṃskṛta→संस्कृत`  
  IASTをデーヴァナガリーに
+ `saṃskṛta→saMskRta`  
  IASTを京都・ハーバード方式に

※ 「サンスクリット変換」のIAST方式、HK方式の変換では拡張IAST（ISO-15919方式）ではなく元来の方式のためISO-15919方式を正しく変換できません。

### 文字列操作

#### 抽出/排除

チェックした文字種類のみを抽出、あるいは排除した文字列を出力する

#### 一括排除

該当の対象文字を文字列から排除する

+ `全ての改行`  
  すべての改行、改頁コードを削除
+ `ＨＴＭＬタグ`  
  開始タグ、終了タグ、独立タグをすべて削除
+ `余分なスペース`  
  行頭のスペースを削除し、単語間の複数スペースを単独スペースに変換
+ `余分な改行`  
  連続する改行コードを単独に変換、文頭文末の改行も削除
+ `重複する文字`  
  重複する文字をすべて削除
+ `重複する行`  
  まったく同じ行を削除
+ `重複する単語`  
  まったく同じ単語を削除（スペース区切り評価）

#### 反転/ソート

文字列全体に対して反転やソートを行う。

+ `文字単位で反転`  
  文字単位でテキスト全体を反転
+ `行単位で反転`  
  行単位でテキスト全体を反転
+ `行ごとに文字単位で反転`  
  一行ずつ文字を反転する
+ `単語単位で反転`  
  単語（スペース区切り評価）でテキスト全体を反転
+ `行単位で昇順に整列`  
  行単位でテキスト全体を昇順ソート
+ `行単位で降順に整列`  
  行単位でテキスト全体を降順ソート
+ `文字単位で昇順に整列`  
  文字単位でテキスト全体を昇順ソート
+ `文字単位で降順に整列`  
  文字単位でテキスト全体を降順ソート
+ `行単位で文字数が少ない順`  
  行ごとの文字数を比べ、少ない順番に並べなおします。
+ `行単位で文字数が多い`  
  行ごとの文字数を比べ、多い順番に並べなおします。

※ ソート順位の評価はUnicode順となります。  
※ ソートの評価値に優劣がつけられない場合、基本的に変換前テキスト先に書かれていたものが先にきます。

#### 重複文字/行列

重複文字に対して操作や行列に対しての変換を行う。

+ `ABA→AB`  
  重複部分のみ削除
+ `ABA→AA`  
  重複部分を重複したまま抽出
+ `ABA→B`  
  重複した文字以外を抽出
+ `ABA→A`  
  重複部分を重複させないで抽出
+ `行と列を入れ替え`  
  チェックされた文字を列区切りに使って行と列を入れ替えます

#### 整形

+ `文字数による改行`  
  変換前テキストから改行をすべて削除した上で、指定された文字数ごとに改行します。
+ `文字の頭出し`  
  変換前テキストから改行をすべて削除した上で、指定した正規表現に該当する文字列の前で改行します。
+ `文字送り`  
  変換前テキストから改行をすべて削除した上で、指定した文字記号のあとで改行します。  
  __「」__ 、 __（）__ については、それぞれの括弧に挟まれた文を一行とみなして前後で改行します。
+ `列挙パスをツリー化`  
  フルパスで書かれたパスをディレクトリごとにツリー化整形します。
+ `ツリーを列挙パス化`  
  ディレクトリごとにツリー表記されたパスをフルパスの列挙型に変換します。

※「列挙パスをツリー化」ではディレクトリを「/（スラッシュ）」で判断します。  
※「ツリーを列挙パス化」では、行末に「/（スラッシュ）」がついた文字列をディレクトリ名と判断し、行頭のタブ文字の数でディレクトリの改装を判断します。

#### 正規表現

正規表現置換を行う

#### 左右連結

左右のテキストエリアを使った連結を行います。

+ `上下に結合`  
  左右のテキストエリアを上下結合した結果を出力
+ `行ごとに結合`  
  左右のテキストエリアを行ごとに結合した結果を出力
+ `文字ごとに結合`  
  左右のテキストエリアを文字ごとに結合した結果を出力

### 文字列生成

#### リピート

入力した文字列を __times__ の回数、リピートします。  
リピートされた文字列は __join__ の正規表現に従って変換前テキストに挿入されます。  
このとき __Reg__ の正規表現によってリピート文字列と変換前文字列の整形ができます。  
`$1`はリピート文字列、`$2`は変換前テキストを示します。

#### 連番

連番を生成します。  
生成される連番は __from__ から __to__ までの間、 __step__ 刻みの数字で、  
__padding__ によってゼロパディングを行います。  
stepが小数点の場合、小数点パディングは自動で行われます。  
生成された連番は __join__ に従って変換前テキストに挿入されます。例えばjoinが  

```
\n
```

のとき、連番は行ごとに挿入されます。

__Reg__ の正規表現に従って連番と変換前テキストが成形されます。  
__overflow__ にチェックがついていると、連番の数が変換前テキストより勝っていても最後まで連番を出力します。  
`$1`は連番、`$2`は変換前テキストを示します。

例えばJoinが `\n` Regが

```
$1: $2
```

の場合、出力されるのは

```
001: 変換前のテキスト1行目
002: 変換前のテキスト2行目
003: 変換前のテキスト3行目
```

といったテキストになります。

※ 計算できないような数値を各値に入れた場合、処理がエラーとなりハングアップする可能性があります。  
※ 小数点第3ケタまで計算します。それ以上の値を入れると小数点に誤差が出る恐れがあります。

#### 日付

日付を基にした連番を生成します。  
生成される連番は __Start__ から __End__ までの間、 __step__ 刻みの日数で、  
__Type__ によって日付パターンを生成します。  
stepが小数点の場合、小数点パディングは自動で行われます。  
生成された日付連番は __join__ に従って変換前テキストに挿入されます。

| パターン | 説明 | 結果（例） |
|:-|:-|:-|
| YYYY | 西暦（4ケタ） | 2018 |
| YY | 西暦（2ケタ）　|　18　|
| MM | 月（ゼロパディング）　|　01　|
| M  | 月　|　1　|
| DD | 日（ゼロパディング）　|　01　|
| D  | 日　|　1　|
| d1 | 曜日（英語）　|　Monday　|
| d2 | 曜日（英語略）　|　Mon　|
| d3 | 曜日（英語1文字）　|　M　|
| d4 | 曜日（日本語）　|　月曜日　|
| d5 | 曜日（日本語1文字）　|　月　|
| m1 | 月（英語） | January |
| m2 | 月（英語略） | Jan |
| m3 | 月（日本語） | 一月 |
| m4 | 月（日本語異名） | 睦月 |

例えば、Joinが `\n` で、Typeが

```
YYYY/MM/DD (d2)
```

Regが

```
$1: $2
```

の場合、出力テキストは

```
2017/01/01 (Sun): 変換前のテキスト1行目
2017/01/02 (Mon): 変換前のテキスト2行目
2017/01/03 (Tue): 変換前のテキスト3行目
・・・・・
```

といった感じになります。

#### 連続データ

五十音やいろは順など、数字ではない連続データを生成する。  
生成される連続データは __amount__ の数値だけ、 __step__ 刻みの数で生成します。  

#### 入力支援

キーボードで入力しにくい文字を入力するための機能です。

+ `符号付英字`  
  アルファベットから符号付アルファベットを検索し入力できます。  
  __Aa__ ボタンを押すと大文字と小文字が切り替わります。
+ `英合字`  
  アルファベットから英字の各種合字を検索し入力できます。
+ `ピンイン入力`  
  アルファベットによりピンインを検索します。
